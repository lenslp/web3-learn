## 测试
1. e2e测试
    + 端到端测试：模拟用户操作，测试整个应用的功能和流程是否正常
2. unit测试
    + 单元测试：测试代码中的最小可测单元（函数、类、组件等），确保其按预期工作
3. 方法论
    + TDD：先写测试 → 测试失败 → 写最少代码让测试通过 → 重构 → 重复
    + BDD：TDD 的"人性化版本"，用“业务行为”的语言描述系统应该做什么
```typescript
// TDD
expect(sum(1, 2)).toBe(3)

// 实现
function sum(a, b) {
  return a + b
}

// BDD
Given 用户已登录
When 用户点击「下单」
Then 订单创建成功
And 跳转到订单详情页
Given('用户已登录', async () => {...})
When('用户点击下单', async () => {...})
Then('订单创建成功', async () => {...})
```
4. 断言库
    + assert：标准库，最底层、最原始，Node.js 内置断言模块
    + chai：专业断言库，功能最全，通常配合 Mocha
    + expect：测试框架内置（jest/Vitest/Playwright），体验最好（主流选择）
        - 框架内置，无需额外安装
        - 错误信息友好
        - 原生异步支持
        - 生态完善
5. 测试框架
    + jest：单元/集成测试，全家桶型
    + mocha：单元/集成测试，配置灵活

    + selenium-webdriver：老牌 E2E 测试工具
    + cypress：现代化的前端端到端（E2E）测试框架，专为 Web 应用设计
    + playwright：现代 E2E / UI 自动化
