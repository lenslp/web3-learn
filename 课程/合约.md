## truffle
1. 是什么
    + 基于 Node.js 的 智能合约开发全流程框架，它整合了 “编译、部署、测试、调试、交互” 等所有开发环节。
1. 常用命令
    + 初始化新 Truffle 项目：truffle init
    + 从模板创建项目：truffle unbox <模板名称>
    + truffle create <contractName> // 创建合约、
    + truffle create all // 创建所有
    + truffle compile // 编译合约
    + truffle migrate // 部署合约到本地 ganache 网络
    + truffle migrate --network sepolia // 部署合约到 sepolia 测试网
    + truffle test // 测试合约

## ganache
1. 是什么
    + 本地以太坊测试网：提供一个本地、离线、快速的以太坊模拟网络，让你不用连接真实的以太坊主网（花钱）或测试网（需要等待区块确认），就能本地调试智能合约

## TypeChain
1. 是什么
    + TypeChain 是一个将以太坊智能合约的 ABI 转换为 TypeScript 类型定义的工具。它通过生成类型安全的合约接口，使开发者能够在 TypeScript 项目中更方便地与智能合约进行交互。
1. 作用
    + 类型安全：TypeChain 生成的类型定义确保在与智能合约交互时，开发者可以获得类型检查和自动补全功能，减少运行时错误。  
    + 提高开发效率：通过提供清晰的类型定义，TypeChain 使得开发者能够更快速地编写和调试与智能合约相关的代码。
    + 支持多种框架：TypeChain 支持多种以太坊开发框架，如 ethers.js 和 web3.js，使其能够与现有的开发工具无缝集成。
