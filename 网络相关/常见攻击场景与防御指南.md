## 1. 公共 WiFi 劫持

### 场景描述
攻击者在公共场所（如咖啡厅、机场）设置伪造的免费 WiFi 热点，诱骗用户连接。

### 攻击原理

```
┌──────────┐
│ 你的设备 │  ← 连接到 "Starbucks_Free_WiFi"
└─────┬────┘
      │
      ↓
┌──────────────┐
│ 攻击者路由器 │  ← 伪造的热点
└──────┬───────┘
      │  记录所有流量
      ↓
┌──────────────┐
│  真实互联网  │
└──────────────┘
```

### 攻击过程
1. 攻击者创建名为"Starbucks_Free_WiFi"的伪造热点
2. 用户连接该热点
3. 攻击者使用中间人攻击（MITM）记录所有网络流量
4. 窃取用户密码、信用卡信息等敏感数据、Cookie

### Web3 风险
- MetaMask 等钱包的种子短语可能被窃取
- DApp 交互的签名请求被截获
- 钱包地址和交易记录被记录

### 防御措施
- ✅ 使用 VPN 加密流量
- ✅ 避免在公共 WiFi 下进行交易或访问钱包
- ✅ 启用钱包的额外安全验证（如密码、生物识别）
- ✅ 确认网络连接的可信度

## 2. DNS 欺骗

### 场景描述
攻击者篡改 DNS 解析结果，将用户导向恶意网站。

### 攻击原理

```
正常访问：
你输入: www.bank.com
     ↓
DNS 返回: 123.45.67.89 (真实银行服务器)

DNS 欺骗：
你输入: www.bank.com
     ↓
被篡改的 DNS 返回: 200.1.2.3 (攻击者的钓鱼网站)
```

### 攻击过程
1. 攻击者侵入 DNS 服务器或使用本地 DNS 缓存投毒
2. 将目标域名的 IP 地址重定向到攻击者控制的服务器
3. 用户访问真实域名，但实际连接到钓鱼网站
4. 用户输入的凭据被窃取

### Web3 风险
- 访问钓鱼 DApp（看起来与真实站点相同）
- 签署恶意交易请求
- 导入攻击者提供的假种子短语

### 防御措施
- ✅ 使用可信的 DNS 服务器（如 1.1.1.1、8.8.8.8）
- ✅ 启用 DNSSEC 验证
- ✅ 仔细检查 URL 和域名
- ✅ 使用硬件钱包进行交易签名
- ✅ 书签保存常用 DApp 网址

## 3. ARP 欺骗

### 场景描述
攻击者在局域网内发送伪造的 ARP 包，将流量重定向到自己的设备。

### 攻击原理

```
局域网内：
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│ 你的电脑        │     │ 路由器          │     │ 攻击者          │
│ 192.168.1.100  │     │ 192.168.1.1     │     │ 192.168.1.50    │
└────────┬────────┘     └────────┬────────┘     └────────┬────────┘
         │                      │                      │
         │                      │                      │
         └──────────────────────┘                      │
                          ↑                            │
                          │                            │
                   正常通信                           │
                                                    ↓
攻击者发送伪造 ARP 包：
"嘿，192.168.1.100，我才是路由器！"

结果：你的流量全部发给攻击者
```

### 攻击过程
1. 攻击者发送伪造的 ARP 响应包
2. 将路由器的 IP 地址映射到攻击者的 MAC 地址
3. 用户流量经过攻击者设备
4. 攻击者可以监听、篡改或窃取数据

### Web3 风险
- 篡改 DApp 的 JavaScript 代码
- 注入恶意交易请求
- 窃取钱包凭据和会话 Cookie

### 防御措施
- ✅ 使用静态 ARP 条目绑定关键设备
- ✅ 启用交换机的 ARP 防护功能
- ✅ 使用 VPN 创建加密隧道
- ✅ 避免在不可信网络中进行 Web3 操作

## 4. SSL 剥离攻击

### 场景描述
攻击者拦截用户与服务器之间的 HTTP 连接，剥离 SSL/TLS 加密。

### 攻击原理

```
正常流程：
┌────────┐    https (安全)    ┌────────┐
│  用户  │ ←─────────────────→ │ 银行   │
└────────┘                      └────────┘

SSL 剥离攻击：
┌────────┐   http (不安全)   ┌────────┐   https (安全)   ┌────────┐
│  用户  │ ←───────────────→ │攻击者  │ ←─────────────→ │  银行  │
└────────┘                     └────────┘                   └────────┘
   ↓                            ↓                            ↓
你以为安全                攻击者在监听               银行以为是安全连接
实际不安全
```

### 攻击过程
1. 用户访问 `http://bank.com`（注意不是 https）
2. 攻击者作为中间人，连接到真实服务器（使用 https）
3. 服务器返回的内容通过 https 传给攻击者
4. 攻击者将内容通过 http 传给你
5. 你以为是安全连接，实际上攻击者在监听

### Web3 风险
- DApp 的交互被监听
- 交易签名请求被截获
- 钱包与节点之间的通信被篡改

### 防御措施
- ✅ 确保访问使用 HTTPS 的网站
- ✅ 浏览器安装 HTTPS Everywhere 插件
- ✅ 启用 HSTS（HTTP Strict Transport Security）
- ✅ 使用 VPN 保护所有流量
- ✅ 在 Web3 操作中使用硬件钱包

## 5. Web3 相关的攻击场景

### 5.1 钓鱼攻击

```
常见钓鱼方式：
┌─────────────────────────────────────────┐
│ 1. 假空投网站                            │
│    "免费领取代币，连接钱包"               │
├─────────────────────────────────────────┤
│ 2. 假 DApp 域名                         │
│    app.uniswap.com → app.unlswap.com    │
├─────────────────────────────────────────┤
│ 3. 社交工程                             │
│    私信假装客服，要求提供种子短语         │
├─────────────────────────────────────────┤
│ 4. 恶意合约授权                         │
│    "授权无限代币" 窃取资产               │
└─────────────────────────────────────────┘
```

### 5.2 恶意扩展程序

```
攻击流程：
1. 安装恶意浏览器扩展
   ↓
2. 扩展读取所有页面内容
   ↓
3. 监听钱包操作
   ↓
4. 窃取种子短语或私钥
   ↓
5. 转移所有资产
```

**防御**：
- ✅ 定期审查已安装的扩展
- ✅ 只安装可信来源的扩展
- ✅ 使用专用浏览器进行 Web3 操作

### 5.3 恶意智能合约

```solidity
// 恶意合约示例
contract Malicious {
    function withdraw() public {
        // 尝试重入攻击
    }
    
    fallback() external payable {
        // 在状态更新前再次调用 withdraw
    }
}
```

**防御**：
- ✅ 使用 ReentrancyGuard
- ✅ 遵循 Checks-Effects-Interactions 模式
- ✅ 使用知名安全团队的合约
