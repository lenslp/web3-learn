## wagmi
作用：wagmi 是一个 React 库，用于与以太坊区块链交互。它提供了一组易于使用的钩子（hooks），用于连接钱包、切换网络、添加网络、查询合约状态、发送交易等。
### 有哪些常用的hooks
#### 钱包连接与状态管理
1. useConnect/useDisconnect：用于连接/断开连接钱包
```
  const { connect, connectors, error } = useConnect();
  // 连接MetaMask
  connect({ connector: connectors[0] }); // connectors[0] 通常是Injected连接器
  // 断开连接
  const { disconnect } = useDisconnect();
  disconnect();
```
2. useAccount：用于获取当前连接的钱包账户信息
```
const { address, isConnected, chainId } = useAccount()
```
3. useBalance：用于获取账户余额
```
const { balance, isLoading } = useBalance({
  address: '0x账户地址'
});
```
#### 网络切换与管理
1. useSwitchChain：用于切换钱包连接的网络
```
const { switchChain } = useSwitchChain();
// 切换到 Ethereum 主网
switchChain({ chainId: 1 });
```
2. useAddChain：用于添加自定义网络到钱包
```
const { addChain } = useAddChain();
// 添加 Ethereum 主网
addChain({
  chainId: 1,
  rpcUrls: ['https://mainnet.infura.io/v3/YOUR_INFURA_PROJECT_ID'],
  blockExplorerUrls: ['https://etherscan.io'],
  nativeCurrency: {
    name: 'Ether',
    symbol: 'ETH',
    decimals: 18
  }
});
```
3. useChainId：用于获取当前连接的链ID
```
const { chainId } = useChainId();
```
#### 合约交互
1. useContractRead： 用于调用合约的只读方法（如查询余额、读取状态变量等）
```
const { data: balance, isLoading } = useContractRead({
  address: contractAddress,
  abi: contractABI,
  functionName: 'balanceOf',
  args: ['0x...']
});
```
2. useContractEvents：用于监听合约事件
```
const { events } = useContractEvents({
  address: contractAddress,
  abi: contractABI,
  eventName: 'Transfer',
  fromBlock: 18000000, // 从指定区块开始监听
});
// 监听 Transfer 事件
events.subscribe(event => console.log(event));
```
3. useContractWrite：用于调用合约的写方法（如转账、调用合约函数等）
```
const { write, isLoading, error } = useContractWrite({
  address: contractAddress,
  abi: contractABI,
  functionName: 'transfer',
});
// 调用：转账1 ETH给目标地址
write({
  args: ["0x接收地址", ethers.parseEther("1")] // 参数：接收方+金额（wei）
});
```
4. usePrepareContractWrite：预计算写操作的参数（如 Gas 费估算），优化用户体验（避免点击后等待）
```
const { config } = usePrepareContractWrite({
  address: "0x合约地址",
  abi: erc20ABI,
  functionName: "transfer",
  args: ["0x接收地址", ethers.parseEther("1")],
});
const { write } = useContractWrite(config); // 传入预计算的配置
```
#### 交易状态与历史
1. useWaitForTransactionReceipt：用于等待交易确认，返回交易收据。
```
const { isLoading, data: receipt } = useWaitForTransactionReceipt({
  hash: '0x交易哈希'
});
```
2. useTransactions：用于获取账户的交易历史
```
const { transactions } = useTransactions({
  address: '0x账户地址'
});
```
3. useTransaction：用于获取交易详情
```
const { isLoading, data: transaction } = useTransaction({
  hash: '0x交易哈希'
});
```
4. useSendTransaction：用于发送交易
```typescript
const { sendTransaction, isPending, isSuccess } = useSendTransaction();
// 发送一笔 ETH 转账，当交易被 成功广播到网络 并从钱包发送后， isSuccess 就会变为true
sendTransaction({
  to: '0x接收地址',
  value: ethers.parseEther('0.1')
});
```
## RainbowKit
+ 作用：专注于钱包连接的 UI 组件库，可以自动处理钱包连接、断开、账户切换

## SWR
+ 作用：用于数据请求的 react hooks库
  + 极速、轻量、可重用
  + 缓存数据
  + 去除重复请求
  + 支持自动刷新数据、聚焦时自动刷新数据
  + 支持 SSR、SSG、ISR
  + 支持 TypeScript

## react-window
+ 作用：用于处理大数据量的列表、表格等，避免渲染所有数据导致性能问题
+ 常用组件：FixedSizeList、VariableSizeList、InfiniteLoader

## rxjs
+ 作用：用于处理异步事件流的库，如事件监听、定时器、HTTP 请求等
+ 优点：
  - 统一异步编程模型：将事件、定时器、API 请求等统一为 “数据流”，逻辑更一致。
  - 强大的操作符：通过链式操作符简化复杂逻辑（如防抖、重试、组合请求）。
  - 通过 unsubscribe() 取消订阅，避免内存泄漏（尤其重要）。
  - 声明式编程：关注 “做什么” 而非 “怎么做”，代码可读性更高。
+ 典型应用场景
  - 事件监听：如用户点击、滚动等
  ```
  import { fromEvent } from 'rxjs';

  // 监听按钮点击事件
  const button = document.querySelector('button');
  const click$ = fromEvent(button, 'click');

  click$.subscribe(() => console.log('按钮被点击了'));
  ```
  - 定时器：如轮询、倒计时等
  ```
  import { interval } from 'rxjs';

  // 每 1 秒触发一次事件
  const interval$ = interval(1000);

  interval$.subscribe(() => console.log('每 1 秒触发一次'));
  ```
  - HTTP 请求：如异步数据获取、批量请求等
  ```
  import { fromFetch } from 'rxjs/fetch';

  // 发送 GET 请求
  const request$ = fromFetch('https://api.example.com/data');

  request$.subscribe(response => {
    if (response.ok) {
      return response.json();
    } else {
      throw new Error('Network response was not ok');
    }
  }).subscribe(data => console.log(data));
  ```
  - 链式操作符：可以对数据流进行多个操作，如过滤、映射、合并等
  ```
  import { fromEvent } from 'rxjs';
  import { map, filter } from 'rxjs/operators';

  // 监听按钮点击事件，过滤出偶数点击，映射为点击次数
  const button = document.querySelector('button');
  const click$ = fromEvent(button, 'click');

  click$
    .pipe(
      filter(() => event.target.tagName === 'BUTTON'), // 过滤出按钮点击事件
      map(() => console.log('按钮被点击了')) // 映射为点击次数
    )
    .subscribe(count => console.log(count));
  ```
  - 复杂表单处理：如表单验证、动态字段添加等

  ```
  // 1. 创建各字段的输入流（带防抖，避免频繁验证）
    const name$ = fromEvent(nameRef.current, 'input').pipe(
      debounceTime(300),
      map(event => event.target.value.trim()),
      distinctUntilChanged(), // 值不变时不触发
      startWith('') // 初始值
    );
  ```

## workbox
+ Google 开发的用于渐进式 Web 应用（PWA）的工具库，主要用于处理 Service Worker 相关功能
+ 作用：
  + 静态资源缓存 ：缓存 JS、CSS、图片、字体等静态资源，提升页面加载速度
  + API 请求缓存 ：缓存动态请求，支持多种缓存策略
    - NetworkFirst：优先从网络获取数据，失败后从缓存中获取
    - CacheFirst：优先从缓存中获取数据，失败后从网络获取
    - StaleWhileRevalidate：先返回缓存数据的，同时请求网络，更新缓存
  + 离线支持 ：在用户离线时，也能正常访问应用
  + 后台同步 ：在用户重新连接网络后，自动同步数据
