## monorepo 管理工具
1. lerna
    + 解决 “多个包怎么发版/发布”（version/publish + 多包管理）
    + 结合workspaces，统一管理依赖
2. turborepo
    + 解决 “monorepo 跑任务太慢”（构建编排 + 增量构建缓存，不负责打包）
    + 处理多包之间的任务依赖关系，先 build 上游，再 build 下游
    + 只跑受影响的包
    + 构建缓存（本地缓存 + 远程缓存）
    + 结合workspaces，统一管理依赖
    + turbo run build：
        - 执行所有子包中的 build 脚本（Turbo 会确保 先构建依赖项，再构建依赖它的包）
        - 检查每个包的输入（源码文件、环境变量等）是否发生变化，只有当输入发生变化时，才会重新构建该包
        - 利用你的 CPU 多核优势， 并行 运行那些互不依赖的任务，从而加快整体构建速度
3. rush
    + 解决 “超大 monorepo 的工程治理”（依赖策略 + 变更管理 + 构建流程标准化）
4. workspaces（npm/yarn/pnpm）
    + 解决 在一个仓库里，如何“安装、链接、管理多个包的依赖关系
    + 统一依赖安装，所有 packages/apps 共享一个 lockfile，确保依赖版本一致、CI / 本地环境结果一致
    + 避免重复安装依赖，节省时间和磁盘空间
    + 自动创建软链接，方便在不同包之间共享代码
5. changesets
    + 自动化版本管理
    + 生成 Changelog
