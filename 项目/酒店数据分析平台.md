## 项目背景
酒店数据分析平台，旨在帮助酒店运营方实时监控多平台（携程、美团等）的房间状态、销售数据和定价信息。通过数据采集、标准化处理和智能分析，为酒店运营人员提供决策支持，优化房间分配和定价策略，提升运营效率和收益水平。
## 技术栈：
nextjs、Ant Design、Tailwind CSS、Recharts、Prisma、PostgreSQL、next-pwa、vercel
## 项目挑战：
+ 不同OTA平台的数据格式差异大，需要设计灵活的数据模型和转换规则
+ 确保房间状态和价格数据的及时性与准确性
+ 优化查询性能，处理大量历史数据的存储和检索
+ 设计可配置的告警规则，及时发现并通知异常情况
## 职责与实现
+ 采用nextjs进行前后端开发，Prisma ORM实现数据库抽象层
+ 设计了完整的数据模型，包括平台管理、房间状态数据、映射规则等11个核心数据实体
+ 实现了酒店名称和房间类型的标准化映射系统，解决多平台数据一致性问题
+ 数据收集脚本开发，支持定时和手动触发
+ 设计并实现了可配置的告警系统，支持多条件组合和不同严重程度的告警
+ 集成企业微信通知系统，实现即时告警推送
## 工作成果
+ 实时监控不同平台相同房型的状态和销售数据，帮助酒店运营人员及时调整运营策略
+ 通过可配置的告警规则，实现异常情况的自动识别和通知，减少人工监控成本
