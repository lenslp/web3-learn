## 数据流重构
### 背景
之前采用的发布订阅模式：
+ 事件订阅分散，不易追踪来源
+ 状态分散、难以维护
### 解决
1. 使用rxjs，把 WebSocket 数据流、接口请求流统一抽象为 Observable（可观察对象）
2. Observable 有生命周期（subscribe/unsubscribe），可以清晰追踪数据来源
3. 支持对高频数据的节流、去重


## 高并发模式下ssr渲染速度慢
高峰期（比如行情剧烈波动）瞬间访问量暴涨，服务器压力可能瞬间爆炸，如果页面 SSR 过程中需要访问行情 API、用户账户数据或订单数据，高并发下 API 响应可能变慢。页面首屏渲染时间反而会增加。
