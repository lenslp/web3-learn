# 浏览器 vs Node.js 事件循环区别

> 创建日期：2026-02-21  
> 分类：JavaScript 基础 / 事件循环

---

## 一句话理解

| 环境 | 事件循环 |
|------|----------|
| **浏览器** | 渲染线程 ↔ JS线程 配合，**每轮宏任务后都可能渲染** |
| **Node.js** | 多个阶段，**微任务在阶段之间执行**，不直接渲染 |

### 浏览器事件循环特点

```
┌─────────────────────────────────────────────────────────────┐
│                    一轮事件循环 =                           │
│                                                             │
│  1. 从宏任务队列取一个执行 (setTimeout, setInterval)        │
│         ↓                                                   │
│  2. 执行过程中产生的微任务入队 (Promise)                    │
│         ↓                                                   │
│  3. 执行所有微任务 (清空微任务队列)                         │
│         ↓                                                   │
│  4. 可能渲染 (requestAnimationFrame → layout → paint)      │
│         ↓                                                   │
│  5. 下一轮循环                                             │
└─────────────────────────────────────────────────────────────┘
```

### Node.js 事件循环特点

```
┌─────────────────────────────────────────────────────────────┐
│                    事件循环阶段 =                            │
│                                                             │
│  ┌──────────┐   ┌──────────┐   ┌──────────┐              │
│  │  Timers   │ → │Pending CB│ → │ idle,    │ → ...        │
│  │ 定时器    │   │  待调用的  │   │ prepare   │              │
│  └──────────┘   └──────────┘   └──────────┘              │
│       ↑                                              │       │
│       │           ┌──────────┐                        │       │
│       └─────── ← │ poll     │ ← setImmediate        │       │
│                   │ I/O轮询  │                        │       │
│                   └──────────┘                        │       │
│                        │                                │       │
│                        ↓                                │       │
│                   ┌──────────┐                        │       │
│                   │  check   │ ← 定时器               │       │
│                   │ 立即执行 │                        │       │
│                   └──────────┘                        │       │
│                        │                                │       │
│                        └─────────── ← ─────────────────┘       │
│                                                             │
│  ⚠️  微任务在每个阶段之间执行，不是在宏任务后！           │
│  ⚠️  Promise.then() 在 Node 中不是立刻执行               │
└─────────────────────────────────────────────────────────────┘
```

## 核心区别（白板简图）

```
                    ┌─────────────────────────────────┐
                    │         浏览器 vs Node.js        │
                    └────────────────┬────────────────┘
                                     │
          ┌──────────────────────────┼──────────────────────────┐
          │                          │                          │
          ▼                          ▼                          ▼
┌─────────────────────┐   ┌─────────────────────┐   ┌─────────────────────┐
│      浏览器          │   │      Node.js         │   │       共同点        │
├─────────────────────┤   ├─────────────────────┤   ├─────────────────────┤
│ 单线程 + Web APIs   │   │ 单线程 + libuv     │   │ 都是单线程          │
│                     │   │                     │   │                     │
│ 执行栈 → 任务队列   │   │ 6个阶段循环        │   │ 都有宏任务/微任务   │
│ (Task Queue)        │   │ (Phase Loop)       │   │                     │
│                     │   │                     │   │ 都是事件驱动        │
│ 宏任务 → 微任务    │   │ 微任务在阶段之间   │   │                     │
│                     │   │                     │   │ 异步处理流程       │
│ 每轮可能渲染        │   │ 不直接渲染         │   │                     │
│ (requestAnimation) │   │                     │   │                     │
└─────────────────────┘   └─────────────────────┘   └─────────────────────┘
```

## 简单对比表

| 特性 | 浏览器 | Node.js |
|------|--------|---------|
| **执行模型** | 单线程 + Web APIs | 单线程 + libuv |
| **任务队列** | 1个宏任务队列 | 6个阶段 (Phases) |
| **微任务执行时机** | 每个宏任务后 | 每个阶段之间 |
| **渲染支持** | ✅ 有 (rAF) | ❌ 没有 |
| **定时器精度** | ≥4ms | ≥1ms |
| **nextTick** | ❌ 没有 | ✅ 有 |
| **setImmediate** | ❌ 没有 | ✅ 有 |

## 经典面试题

### 题1: 输出顺序

```javascript
console.log('1');

setTimeout(() => console.log('2'), 0);

Promise.resolve().then(() => console.log('3'));

console.log('4');
```

**浏览器输出**: 1 → 4 → 3 → 2  
**Node.js输出**: 1 → 4 → 3 → 2  
(两者相同，因为 Promise 是微任务)

### 题2: setTimeout vs setImmediate

```javascript
setTimeout(() => console.log('timeout'), 0);
setImmediate(() => console.log('immediate'));
```

**浏览器输出**: timeout → immediate  
**Node.js输出**: 不一定！可能是 timeout → immediate 或 immediate → timeout  
(取决于 I/O 状态)

### 题3: process.nextTick

```javascript
Promise.resolve().then(() => console.log('promise'));
process.nextTick(() => console.log('nextTick'));
```

**Node.js输出**: nextTick → promise  
(因为 nextTick 在当前阶段立即执行，优先于微任务)

## 图解：浏览器完整流程（白板版）

```
┌──────────────────────────────────────────────────────────────────┐
│                         浏览器事件循环                            │
└──────────────────────────────────────────────────────────────────┘

   用户操作/定时器/网络请求
              │
              ▼
   ┌───────────────────────┐
   │       执行栈          │ ← 同步代码在这里执行
   │    (Call Stack)       │
   └───────────┬───────────┘
              │
              │ 调用 Web API
              ▼
   ┌───────────────────────┐
   │      Web APIs         │ ← setTimeout, DOM, fetch 等
   │   (浏览器提供)        │
   └───────────┬───────────┘
              │
              │ 完成时
              ▼
   ┌───────────────────────┐
   │     任务队列          │ ← 宏任务回调
   │  (Task Queue)         │   setTimeout / setInterval / I/O
   └───────────┬───────────┘
              │ 进入事件循环
              ▼
   ┌───────────────────────┐
   │    事件循环检查       │ ◄── 每一轮
   │   (Event Loop)        │
   └───────────┬───────────┘
              │
              │ 取出一个宏任务执行
              ▼
   ┌───────────────────────┐
   │      执行栈           │ ← 执行这个宏任务
   └───────────┬───────────┘
              │
              │ 产生微任务
              ▼
   ┌───────────────────────┐
   │      微任务队列        │ ← Promise.then /MutationObserver
   │   (MicroTask Queue)  │
   └───────────┬───────────┘
              │ 执行所有微任务
              │ (直到队列为空)
              ▼
   ┌───────────────────────┐
   │      可能渲染         │ ◄── requestAnimationFrame
   │   (Render)           │     layout → paint
   │                      │     (如果需要)
   └───────────────────────┘
              │
              │ 下一轮循环
              └─────────────── → (回到"取出一个宏任务")
```

## 图解：Node.js 完整流程（白板版）

```
┌──────────────────────────────────────────────────────────────────┐
│                        Node.js 事件循环                          │
└──────────────────────────────────────────────────────────────────┘

        输入/定时器/I/O/关闭事件
                    │
                    ▼
   ┌───────────────────────┐
   │   ┌─ Timer ──────────┐│ ← setTimeout / setInterval
   │   │                   │
   │   ├─ Pending Callback ││ ← 内部回调
   │   │                   │
   │   ├─ idle, prepare    ││ ← 内部使用
   │   │                   │
   │   ├─ Poll ───────────││ ← I/O 回调 / fetch
   │   │                   │
   │   ├─ Check ───────────││ ← setImmediate
   │   │                   │
   │   └─ Close callbacks  ││ ← socket.close
   └───────────────────────┘
                    │
                    │ 进入事件循环
                    ▼
   ┌───────────────────────┐
   │      Timer 阶段       │ ◄── 执行到期的 setTimeout/setInterval
   └───────────┬───────────┘
              │
              │ 微任务 (nextTick > microtask)
              ▼
   ┌───────────────────────┐
   │   Pending Callback    │ ◄── 执行内部回调
   └───────────┬───────────┘
              │
              │ 微任务
              ▼
   ┌───────────────────────┐
   │   Poll 阶段           │ ◄── I/O 相关回调
   └───────────┬───────────┘
              │
              │ 有 setImmediate?
              │    ↓ 是 → 跳到 Check 阶段
              │    ↓ 否
              │ 有到期的定时器?
              │    ↓ 是 → 回到 Timer 阶段
              │    ↓ 否 → 阻塞等待 I/O
              ▼
   ┌───────────────────────┐
   │    Check 阶段         │ ◄── setImmediate 回调
   └───────────┬───────────┘
              │
              │ 微任务
              ▼
   ┌───────────────────────┐
   │  Close Callbacks     │ ◄── 关闭回调
   └───────────┬───────────┘
              │
              │ 下一轮
              └─────────────── → (回到 Timer 阶段)
```

## 常见误区

### ❌ 误区1: Node.js 没有微任务
✅ 错！Node.js 也有 Promise 微任务，只是执行时机不同

### ❌ 误区2: setTimeout(fn, 0) 立即执行
✅ 错！浏览器最小 4ms，Node 最小 1ms

### ❌ 误区3: 两者完全相同
✅ 错！微任务执行时机不同，渲染机制不同
