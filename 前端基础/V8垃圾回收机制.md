# V8 垃圾回收机制

> 创建日期：2026-02-21  
> 分类：JavaScript 基础 / V8 引擎 / 性能优化

---

## 一句话理解

**V8 垃圾回收 = 自动内存管理，把不再使用的对象清理掉，释放内存空间。**

## 垃圾回收算法（白板图）

### 1. Scavenge - 新生代垃圾回收

```
┌─────────────────────────────────────────────┐
│              Scavenge (新生代)              │
│                                             │
│  适用：新生代区 (From 空间)                 │
│  原理：标记 → 复制 → 翻转                  │
└─────────────────────────────────────────────┘

步骤1: 标记
┌────────────┐
│  From 空间  │
│  ┌──┐ ┌──┐ │  ← 存活对象
│  │A │ │B │ │
│  └──┘ └──┘ │
│  ┌──┐      │  ← 死亡对象 (无引用)
│  │C │      │
│  └──┘      │
└────────────┘

步骤2: 复制存活对象到 To 空间
┌────────────┐
│  To 空间   │
│  ┌──┐ ┌──┐ │
│  │A │ │B │ │  ← 复制存活对象
│  └──┘ └──┘ │
└────────────┘

步骤3: 翻转 From 和 To
┌────────────┐     ┌────────────┐
│  From ←新  │ ←   │  To  ←旧  │
│  (空闲)    │     │  (使用中)  │
└────────────┘     └────────────┘
```

### 2. Mark-Sweep - 标记清除（老生代）

```
┌─────────────────────────────────────────────┐
│           Mark-Sweep (老生代)              │
│                                             │
│  适用：老生代区                             │
│  原理：标记 → 清除 → 整理                   │
└─────────────────────────────────────────────┘

步骤1: 标记 (从根对象出发)
┌────────────┐
│   堆内存    │
│  ┌──┐ ┌──┐ │  ← 存活 (有引用)
│  │A │ │B │ │
│  └──┘ └──┘ │
│  ┌──┐      │  ← 死亡 (无引用)
│  │C │      │
│  └──┘      │
└────────────┘

步骤2: 清除 (删除死亡对象)
┌────────────┐
│   堆内存    │
│  ┌──┐ ┌──┐ │  ← 保留
│  │A │ │B │ │
│  └──┘ └──┘ │
│     ┌───┐   │  ← 变成碎片
│     │   │   │
│     └───┘   │
└────────────┘
```

### 3. Mark-Compact - 标记整理（解决内存碎片）

```
┌─────────────────────────────────────────────┐
│          Mark-Compact (标记整理)           │
│                                             │
│  目的：解决内存碎片问题                      │
│  原理：标记 → 整理 → 清除                   │
└─────────────────────────────────────────────┘

整理前:
┌────────────┐
│ A │ B │ C │ D │ E │   ← 存活对象，分散在各处
└────────────┘

整理后:
┌────────────┐
│ A │ B │ C │ D │ E │   ← 存活对象连续排列
└────────────┘
      ↓
┌────────────┐
│ 可用空间    │   ← 整理出连续空间
└────────────┘
```

┌─────────────────────────────────────────────────────────────┐
│                    V8 垃圾回收完整流程                       │
└─────────────────────────────────────────────────────────────┘

                          ┌──────────────────┐
                          │ 1. JavaScript 执行│
                          │    产生新对象     │
                          └────────┬─────────┘
                                   │
                                   ▼
┌─────────────────────────────────────────────────────────────┐
│                    2. Scavenge (新生代)                      │
│                                                             │
│   From 空间 ──复制存活对象──→ To 空间                      │
│   (存活少，死亡多效率高)                                      │
│                                                             │
│   ┌─────────┐      ┌─────────┐                            │
│   │ From    │  →   │  To     │                            │

│   └─────────┘      └─────────┘                            │
└─────────────────────────────────────────────────────────────┘
                                   │
              ┌──────────────────────┼──────────────────────┐
              │                      │                      │
              ▼                      ▼                      ▼
    ┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐
    │ 对象存活久      │  │  对象过大       │  │  其他对象       │
    │ (晋升老生代)    │  │ (直接老生代)    │  │  (留在新生代)   │

                                   │
                                   ▼
┌─────────────────────────────────────────────────────────────┐
│                  3. Mark-Sweep + Mark-Compact              │
│                    (老生代垃圾回收)                          │
│                                                             │
│   步骤: 标记 → 整理 → 清除                                  │
│                                                             │
│   根对象                                                    │
│      │                                                      │
│      ▼                                                      │
│   ┌─────────┐      ┌─────────┐      ┌─────────┐         │
│   │  标记   │  →   │  整理   │  →   │  清除   │         │
│   │ 存活对象│      │  移动对象│      │ 删除死亡│         │
│   └─────────┘      └─────────┘      └─────────┘         │
└─────────────────────────────────────────────────────────────┘
                                   │
                                   ▼

## 分代回收策略（白板图）

```
┌─────────────────────────────────────────────────────────────┐
│                    为什么需要分代？                          │
│                                                             │
│   💡 经验规律：                                              │
│   - 大多数对象"朝生夕死" (活不过一次垃圾回收)               │
│   - 少数对象存活时间很长                                   │
│                                                             │
│   → 针对不同生命周期对象，用不同回收策略                     │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│                        分代回收                              │
└─────────────────────────────────────────────────────────────┘

  新生代 (Young Generation)          老生代 (Old Generation)
  ┌─────────────────────┐          ┌─────────────────────────┐
  │  特点：              │          │  特点：                │
  │  - 对象存活时间短   │    晋升   │  - 对象存活时间长     │
  │  - 98%朝生夕死     │ ──────→  │  - 少数长期存活       │
  │  - 空间小 (1-2MB) │          │  - 空间大 (1400MB)     │
  │                     │          │                         │
  │  回收算法：         │          │  回收算法：            │
  │  Scavenge          │          │  Mark-Sweep            │
  │  (复制算法)         │          │  Mark-Compact          │
  │  (速度快)           │          │  (标记清除/整理)       │
  │                     │          │  (速度慢，全面)        │
  └─────────────────────┘          └─────────────────────────┘
          ↑                                   ↑
          │           对象晋升                  │
          └───────────────────────────────────┘
```

## 常见的 GC 术语

| 术语 | 解释 |
|------|------|
| **GC** | Garbage Collection，垃圾回收 |
| **堆 (Heap)** | V8 存储对象的内存区域 |
| **新生代 (New Space)** | 存放新创建的对象，空间小 |
| **老生代 (Old Space)** | 存放存活久的对象，空间大 |
| **Scavenge** | 新生代回收算法，复制方式 |
| **Mark-Sweep** | 标记清除，老生代常用 |
| **Mark-Compact** | 标记整理，解决内存碎片 |
| **晋升 (Promotion)** | 新生代对象升级到老生代 |
| **根对象 (Roots)** | 全局对象、调用栈等可达起点 |
| **空闲列表 (Free List)** | 记录可用内存块 |

## 面试常见问题

### Q1: V8 为什么用分代回收？

```
┌─────────────────────────────────────────┐
│ 原因：                                            │
│                                                 │
│ 1. 优化性能                                      │
│    - 新生代对象多，死亡快 → 用快算法             │
│    - 老生代对象少，死亡慢 → 用全面算法           │
│                                                 │
│ 2. 经验规律                                      │
│    - 98%对象活不过一次GC                        │
│    - 少数对象存活很久                          │
│                                                 │
│ 3. 空间换时间                                   │
│    - 新生代空间小，复制成本低                   │
│    - 老生代空间大，全量扫描成本高               │
└─────────────────────────────────────────┘
```

### Q2: 什么情况下对象会进入老生代？

```
┌─────────────────────────────────────────┐
│ 晋升老生代的条件：                              │
│                                                 │
│ 1. 新生代回收后仍然存活                       │
│    (经历两次 Scavenge)                        │
│                                                 │
│ 2. To 空间不足时                              │
│    (复制时空间不够)                            │
│                                                 │
│ 3. 大对象直接分配                              │
│    (大于 To 空间 25%)                         │
│                                                 │
│ 4. 构造函数创建                               │
│    (直接从老生代分配)                          │
└─────────────────────────────────────────┘
```

### Q3: 如何避免内存泄漏？

```javascript
// ❌ 常见内存泄漏

// 1. 全局变量
function leak() {
  bigData = new Array(1000000);  // 没有 var/let 声明
}

// 2. 闭包
function closure() {
  let bigData = new Array(1000000);
  return function() {
    console.log(bigData);  // bigData 永远不会被释放
  };
}

// 3. 事件监听未移除
element.addEventListener('click', handler);
element.removeEventListener('click', handler);  // 忘记移除

// 4. 定时器未清除
setInterval(() => {
  // 定时器内的引用
}, 1000);
// 页面关闭时未清除

// ✅ 正确做法

// 1. 使用 let/const
function noLeak() {
  const bigData = new Array(1000000);
}

// 2. 及时移除事件监听
element.addEventListener('click', handler);
element.removeEventListener('click', handler);

// 3. 清除定时器
const timer = setInterval(() => {}, 1000);
clearInterval(timer);

// 4. 使用 WeakMap/WeakSet
const cache = new WeakMap();  // 对象不可达时自动回收
```

### Q4: V8 垃圾回收会阻塞 JS 执行吗？

```
┌─────────────────────────────────────────┐
│  会！但做了优化：                              │
│                                                 │
│  1. 增量标记 (Incremental Marking)           │
│     - 把标记过程拆分成多个小步骤               │
│     - 每执行一小步就暂停                       │
│                                                 │
│  2. 惰性清理 (Lazy Sweeping)                  │
│     - 不立即清理，先返回执行 JS                │
│     │  等空闲时再清理                         │
│                                                 │
│  3. 并行回收 (Parallel Scavenge)             │
│     - 使用多个线程并行回收                      │
│                                                 │
│  4. 写屏障 (Write Barrier)                   │
│     - 记录对象引用变化                         │
│     - 增量标记时跟踪变化                       │
│                                                 │
│  结果：GC 造成的停顿肉眼基本不可见              │
└─────────────────────────────────────────┘
```

## 实际开发建议

### 1. 减少内存占用

```javascript
// ❌ 不好
const arr = [];
for (let i = 0; i < 1000000; i++) {
  arr.push({ index: i, data: new Array(100) });
}

// ✅ 好
class Item {
  constructor(index) {
    this.index = index;
  }
}
// 使用对象池复用
```

### 2. 及时释放引用

```javascript
// ❌
const cache = {};
function add(key, value) {
  cache[key] = value;  // 永远增长
}

// ✅
const cache = new Map();
function add(key, value) {
  if (cache.size > 100) {
    const firstKey = cache.keys().next().value;
    cache.delete(firstKey);
  }
  cache.set(key, value);
}
```

### 3. 使用合适的数据结构

```javascript
// 处理大量数据时
// ❌ 数组操作慢
const arr = [];
arr.push(obj);
arr.find(x => x.id === target);

// ✅ Map 更快
const map = new Map();
map.set(target, obj);
map.get(target);
```
