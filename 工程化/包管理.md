### pnpm相比npm/yarn的优势是什么
+ 全局内容寻址仓库：所有包统一存到全局 store (~/.pnpm-store)，根据包内容计算哈希值，以哈希作为路径/文件名，相同内容的包只存一份，从而实现去重缓存。
+ 硬链接 + 符号链接(软链接)：安装时从 store 建立硬链接把真实文件挂到项目下的.pnpm目录，再用符号链接把这些包挂到 node_modules 顶层。由于硬链接指向同一个底层 inode，不同项目的相同依赖共享同一份物理文件，既节省磁盘，又能加快安装（无需反复拷贝）
+ 严格依赖解析：node_modules 结构让包只能访问自己声明的依赖，避免幽灵依赖，并支持多版本依赖安全共存。

## monorepo
+ 多个项目/模块放在同一个 Git 仓库一起管理的代码组织方式
    - 不再每个模块建一个独立 repo
    - 把所有前端、后端、组件库、工具库放在一个仓库里
    - 用 workspace 机制管理它们的依赖、编译、构建关系

## workspace
+ 在同一个仓库中管理多个互相引用的 package 的机制
    - 一个仓库里可以有多个包（packages）
    - 这些包之间可以直接互相依赖，而不用发到 npm
    - 可以共享 node_modules、共享依赖版本、共享工具链
