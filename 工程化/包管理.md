### pnpm相比npm/yarn的优势是什么
+ 安装速度最快：安装过程中支持并行下载和解压，进一步提升了安装速
+ 占用磁盘空间少：每个依赖包在磁盘上只会存储一次，无论它被多少个项目所依赖，依赖包文件都会指向同一份磁盘数据。（通过 硬链接 在项目的 node_modules 文件夹中创建链接）
+ 更严格，能避免幽灵依赖，不写在 package.json 就不能用，避免线上事故
+ 自带 workspace，适用于monorepo项目

## monorepo
+ 多个项目/模块放在同一个 Git 仓库一起管理的代码组织方式
    - 不再每个模块建一个独立 repo
    - 把所有前端、后端、组件库、工具库放在一个仓库里
    - 用 workspace 机制管理它们的依赖、编译、构建关系

## workspace
+ 在同一个仓库中管理多个互相引用的 package 的机制
    - 一个仓库里可以有多个包（packages）
    - 这些包之间可以直接互相依赖，而不用发到 npm
    - 可以共享 node_modules、共享依赖版本、共享工具链
