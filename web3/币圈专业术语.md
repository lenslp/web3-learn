## KYC认证
KYC 认证是 Know Your Customer（了解你的客户） 的缩写，是金融、支付、加密货币等领域广泛推行的合规制度，核心是要求机构通过收集和验证用户身份信息，确认用户真实身份，从而防范洗钱、恐怖融资、身份盗用、欺诈等违法违规行为。

## DID
DID是未来更具潜力的身份解决方案。在DID体系下，用户的身份并非由某个特定机构颁发，而是由用户自主生成和管理一系列可验证凭证。
+ 代表项目：ENS

## 现货和合约
1. 现货：现买现卖的资产，可长期持有，只要不主动卖出，资产就一直属于所有者，无到期强制平仓风险，没有杠杆，适合长期价值投资，收益计算：(卖出价格 - 买入价格) * 数量 - 交易手续费。
2. 合约：不直接买卖资产，而是交易双方签订的、约定未来以特定价格买卖的协议，自带杠杆功能，适合风险较高的交易。
    + 永续合约（Perpetual Contract）：没有到期日和交割日的合约，投资者可长期持仓，无需担心合约到期强制结算，其价格通过资金费用机制锚定现货价格。
        - 无到期日，适合长期持仓，靠 “资金费用”（每 8 小时结算）让价格贴近现货。
        - 资金费用机制：这是永续合约的核心设计，目的是让合约价格贴近现货价格。通常每 8 小时结算一次，当合约价格高于现货价格时，多头向空头支付资金费用；当合约价格低于现货价格时，空头向多头支付资金费用，通过供需平衡校准价格。
        - 支持高杠杆
    + 交割合约（Delivery Contract）：指约定了固定到期日的合约，到期后合约会按约定方式进行结算（交割），终止持仓关系。
        - 有日期限制，投资者需在到期前主动平仓，或按规则参与交割
        - 没有资金费用，价格波动主要受标的现货价格、市场供需等因素影响
        - 分为实物交割和现金交割。
        - 支持高杠杆

## 前端和合约交互为何要进行approve
approve 操作的核心作用是授权第三方合约（或地址）有限度地操作当前用户的资产。
为什么？
1. 防止恶意合约滥用资产，只有授权的合约才能操作用户资产，approve 允许用户精确控制授权的额度和对象。
2. 区块链上的资产默认只有资产所有者（钱包地址）有权转移或操作。但在实际场景中，用户常需要通过第三方合约（如去中心化交易所、借贷平台）完成复杂操作。

## DeFi 
+ 去中心化金融（Decentralized Finance，简称 DeFi）是指基于区块链技术的金融服务，不依赖于传统金融机构，而是通过智能合约在去中心化网络上运行。DeFi 服务包括但不限于借贷、交易、存款、保险等。
+ 代表项目：Uniswap

## 标的（di）
标的（Underlying Asset）：指在合约交易中，被交易的资产。例如，在 ETH/USDT 交易对中，ETH 是标的资产，USDT 是结算资产。

## 交易对
交易对通常由 “基础标的” 和 “计价标的” 两部分组成，格式一般为「基础标的 / 计价标的」，比如 BTC/USDT、ETH/USD、黄金 / 美元等
+ 基础标的：你想买入或卖出的核心资产（比如比特币、以太坊、黄金）；
+ 计价标的：用于衡量基础标的价值的资产（比如稳定币 USDT、法定货币 USD、其他主流资产）。
场景：
+ 当 BTC/USDT 的价格为 42000 时，你用 42000 USDT 买入 1 枚比特币，本质是完成 “USDT → 比特币” 的兑换；
+ 后续当价格涨到 45000 时，你卖出这 1 枚比特币，得到 45000 USDT，盈利 3000 USDT，整个交易过程围绕 BTC/USDT 这个交易对展开+ 

## 平仓、强制平仓、穿仓
1. 平仓：投资者主动卖掉之前买的，或买回之前卖的”，终止当前的持仓状态
2. 强制平仓：当投资者的持仓账户触发特定风险条件时，交易平台或券商无需投资者同意，自动对其持仓进行平仓处理。
    + 保证金不足：这是最常见的原因。合约交易中，投资者需缴纳一定比例的保证金作为履约担保。当标的价格波动导致账户亏损，使保证金余额低于平台规定的 “维持保证金比例” 时，平台会启动强平
3. 穿仓：亏损超过账户保证金，部分平台会向投资者追偿穿仓损失。

## 多头、空头
1. 多头/做多：指投资者预期标的资产（如股票、加密货币、大宗商品等）价格未来会上涨，从而买入资产并持有，等待价格上涨后卖出获利的持仓状态。（看涨）
2. 空头/做空：指投资者预期标的资产价格未来会下跌，通过 “借入资产卖出” 的方式建立持仓，等待价格下跌后再低价买入资产归还，赚取差价收益的持仓状态。（看跌）

## 撸空投
撸空投本质是区块链生态中，用户通过完成项目方任务免费获取空投代币（Token），再通过交易变现的 “羊毛党” 行为

## 区块确认
区块确认是指一笔交易被打包进新区块后，该区块以及包含这笔交易的后续区块，被全网节点验证并添加到区块链主链的过程
1. 交易发起与广播：用户发起一笔交易（如转账、合约交互），将交易信息（包含发送方、接收方、金额、签名等）广播至全网节点；
2. 区块打包：矿工（PoW 机制）或验证者（PoS 机制）将网络中未确认的交易收集起来，打包成一个新的候选区块，并完成对应的共识验证（如 PoW 的算力解谜、PoS 的质押权益验证）；
3. 首次确认：候选区块通过共识后，被添加到区块链主链的末端，此时包含在该区块内的所有交易完成 “1 次确认”；
4. 多轮确认：随着后续新区块不断被打包并添加到主链，每新增一个后续区块，之前区块内的交易就多一次 “确认”。例如，交易所在区块 A 中，后续生成区块 B、C、D，那么该交易的确认次数就是 4 次（A 自身 + B、C、D 三个后续区块）

## 预言机
+ 是连接区块链网络与链下现实世界 / 外部系统的 “数据桥梁”。由于区块链本身是封闭的分布式账本，无法主动获取链下信息（如现实世界的价格数据、天气情况、赛事结果、API 接口数据等），而智能合约的执行往往依赖这些外部数据，预言机的核心作用就是将链下数据安全、可靠地传输到链上，为智能合约提供触发条件和决策依据。
+ 代表项目：[Chainlink](https://chain.link/)
+ 举个例子： 
    - 数据输入：将链下数据（如加密货币价格、股票指数、汇率、物联网设备数据等）上传至链上，供智能合约调用；
    - 事件触发：基于链下特定事件（如航班延误、地震预警、合约到期提醒等）触发智能合约执行（如保险赔付、衍生品结算）；
    - 结果验证：将智能合约的链上执行结果同步至链下系统（如支付结算通知、物流状态更新），完成闭环交互

## 节点服务
+ 节点服务相当于WEB3领域的云计算服务。
+ 并非每个开发者都愿意自己搭建节点来与区块链进行交互，节点服务提供商就应运而生。这些公司会提供现成的应用程序编程接口，让开发者能够轻松地查询链上数据、发送交易。其代表项目有Infura和Alchemy。

## IPFS
+ 区块链本身不适合存储大量的文件数据，比如图片、视频，主要是太贵
+ IPFS是一种网络协议，它采用内容寻址的方式，即通过文件的哈希值来查找文件，而非像传统方式那样通过文件位置进行查找，其目标是取代HTTP协议
+ IPFS本身并不保证数据存储的永久性，它需要依赖激励层，例如Filecoin。在Filecoin网络中，节点需抵押代币以承诺存储数据

## PINATA
为开发者和用户提供便捷的 IPFS 接入服务，简化文件上传、管理和访问流程。用户可通过 Pinata 将文件上传至其维护的 IPFS 节点，并通过 “Pin（固定）” 功能确保文件不会被节点删除（避免因节点下线导致数据丢失）
+ 访问我上传的文件：
    - https://gateway.pinata.cloud/ipfs/[图片的CID]
    - https://[你的网关域名]/ipfs/[图片的CID]

## Filecoin
+ 通过区块链和代币经济（FIL 代币）解决 IPFS 的 “存储动力” 问题：
+ 存储提供者（矿工）：提供硬盘空间存储用户数据，获得 FIL 奖励；
+ 存储使用者：支付 FIL 雇佣矿工存储数据，并通过智能合约确保数据被持续存储（矿工需质押 FIL，若数据丢失则被罚没）；
+ 数据验证：通过 “复制证明”“时空证明” 等机制，确保矿工真实存储了数据且未篡改

## Arweave
+ 永久存储的去中心化区块链存储网络（独立协议）
+ 数据一旦写入，会被全网节点永久保存（节点无法删除历史数据）。用户支付一次 AR 代币（Arweave 的原生代币），即可让数据在网络中永久留存，适合需要长期归档的数据（如学术文献、历史记录、法律文件）

## IPFS、Pinata、Filecoin、Arweave
IPFS 是底层协议，解决 “如何分布式存储和传输文件”；
Pinata 是 IPFS 的 “易用性工具”，降低接入门槛；
Filecoin 是 IPFS 的 “经济引擎”，通过激励让存储可持续；
Arweave 是 “永久存储的替代方案”，专注不可删除的长期归档。

## NFT
+ NFT（Non-Fungible Token）是一种基于区块链的数字资产，每个 NFT 都是独一无二的，无法被替代或复制。
+ 代表项目：[OpenSea](https://opensea.io/)

## layer0、1、2
1. layer0：区块链网络基础设施，负责跨链通信、网络传输、节点连接等最基础的功能。
2. layer1：我们通常所说的 “区块链主链”，是直接处理交易、维护账本、实现共识机制的核心协议层，负责数据存储、交易验证、共识达成（如 PoW、PoS），并定义代币经济、智能合约规则等核心逻辑，比如以太坊、比特币 等。
3. layer2：建立在 Layer 1 之上的 “扩展解决方案”，目的是解决 Layer 1 的性能瓶颈，将部分交易或计算从 Layer 1 转移到二层处理，仅将最终结果提交给 Layer 1 上链，从而提升整体效率。
4. layer3：建立在 Layer 1/2 之上的 “应用层”，提供具体的业务功能和服务，如 DeFi 应用、游戏、社交网络等。
    - DeFi：uniswap、sushiswap等去中心化交易所
    - Game：基于智能合约的去中心化游戏平台
    - NFT：OpenSea等非同质化代币平台

## 为什么要区分layer层级
1. 区块链的 “不可能三角”（去中心化、安全性、扩展性难以同时满足），通过分层架构，将不同功能模块分配到不同层级，实现 “互补” 而不是 “冲突”。
2. 明确功能边界，实现专业化分工
3. 各层级独立迭代，降低开发门槛
4. 平衡安全性于扩展性，layer1保障安全，layer2提升扩展性

## 头寸
金融领域中常用的一个术语，指的是投资者或交易者在某一特定资产（如股票、期货、外汇、加密货币等）上的持有或交易状态，表示其在市场中所拥有的某一资产的数量、价值以及方向（多头或空头）
+ 多头头寸：指投资者或交易者购买资产的行为，资产的价值上涨时，其持仓价值会增加，资产的价值下跌时，其持仓价值会减少。（看涨）
+ 空头头寸：指投资者或交易者出售资产的行为，资产的价值上涨时，其持仓价值会减少，资产的价值下跌时，其持仓价值会增加。（看跌）
+ 杠杆头寸：通过借入资金进行投资的头寸。这意味着投资者通过杠杆放大了自己的投资，但也增加了潜在的风险。
+ 对冲头寸（对冲头寸）：通过持有反向头寸来减少市场风险。
